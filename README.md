# Government Commodity Price Analysis

## Project Overview
This project analyzes government mandi (market) commodity price data to extract meaningful insights and trends. It simulates a real-world data analytics pipeline using Python, SQL, and AWS S3 for data storage and processing.

---

## Objectives
- Analyze commodity price trends across different markets  
- Identify seasonal patterns and fluctuations  
- Perform data cleaning and transformation  
- Generate insights using SQL and Python  
- Build an end-to-end data pipeline  

---

## Tech Stack
- **Programming:** Python  
- **Database:** MySQL  
- **Cloud:** AWS S3  
- **Libraries:** Pandas, NumPy, Matplotlib, Seaborn  
- **Tools:** Jupyter Notebook / VS Code  

---

## Project Architecture
Data Source (CSV Files)
↓
AWS S3 (Storage Layer)
↓
Python (Data Cleaning & Processing)
↓
MySQL (Data Storage & Querying)
↓
SQL Analysis & Insights
↓
Visualization & Reporting


---

## Key Features
- End-to-end data pipeline implementation  
- Data cleaning and preprocessing using Python  
- SQL-based analysis (joins, aggregations, filtering)  
- Trend and pattern analysis  
- Insight generation for decision-making  

---

## Analysis Performed
- Price trends over time  
- Market-wise comparisons  
- Commodity-wise distribution  
- Seasonal price variations  
- High and low price identification  

---

## Dataset
The dataset includes:
- Commodity names  
- Market locations  
- Arrival quantities  
- Min, max, and modal prices  
- Transaction dates
